Direction
  • Direction

    An instruction for a user to get from one Node to another. It will have a Node it happens at, an Action (depart, arrive, turn, take vortex), and optional Locations to perform this at or towards, as well as a String representing the text instructions to display to the user.

    The at/towardsLocation properties are to let you mark or highlight key reference locations on the MapView if they will help the user find or keep their bearings.

  • Vortex

    A Vortex is a special entity that represents a link between two or more maps at a given venue. It is typically something like an elevator or stairs.

    They can be marked (wheelchair) Accessible or not for pathfinding purposes. If the user has selected Accessible directions, paths that include non-accessible Vortexes like Stairs will not be used.

    A Vortex contains a list of Maps it connects. It is NOT a Location.

  • Node

    A Node represents specific coordinates on a map, used for pathing. A path is just a sequential list of Nodes from one place to another.

    Typically the start and end points of a path will belong to a specific location, but this is not actually required, and the Node itself has no knowledge of who may or may not own it.

    A Node knows what map it belongs to, and it’s coordinates in 3D space (though the Z coordinate will be a constant across all nodes)

  • Venue

    A Venue is a specific place (like a mall) with one or more maps (typically representing different floors) and locations (stores, washrooms, elevators, etc) that belong to the venue and have specific locations on certain maps.

    A Venue can be in two states: 0. getVenues - When it’s initially pulled from the MappedIn API as part of a list of all venues your credentials have access to, it only contains basic information like name, logo and cover image. 0. getVenue - When you retrieve the full details of a given venue, the locations, categories, maps, nodes and vorticies will be populated.

  • Location

    A location is something like a store or washroom on a map. It has a name and type, description, logo, and picture (of the storefront, for example).

    A location can also belong to one or more categories (Store, Jewelery Store, etc), and will be linked to one or more nodes and polygons.

    A location’s nodes can be on multiple maps, or in multiple non-adjacent places on the same map. For example, all washroom at a given venue could belong to one location, or a department store could live on multiple floors.

  • Category

    A Category is a collection of similar Locations, which may have parent or children categories. For example, you may have a Fashion category, which a Men’s Fashion and Women’s Fashion category. A Men’s clothing store would belong to the Men’s Fashion category, but not nessisarily be present in the Fashion category’s list of Locations.
